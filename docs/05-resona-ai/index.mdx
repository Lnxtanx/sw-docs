# Resona AI

Resona is Schema Weaver's built-in AI data assistant. It understands your database schema and lets you explore, query, and analyze your data using natural language — no SQL expertise required.

## What Resona Can Do

### Natural Language to SQL

Ask questions about your data in plain English. Resona generates optimized SQL, explains it step by step, and runs it for you:

> "How many new users signed up each week over the last 3 months?"

```sql
SELECT
  DATE_TRUNC('week', created_at) AS week,
  COUNT(*) AS new_users
FROM users
WHERE created_at >= NOW() - INTERVAL '3 months'
GROUP BY week
ORDER BY week;
```

### Automatic Chart Generation

When results contain numeric or time-series data, Resona automatically suggests and renders the most appropriate visualization — bar charts, line charts, pie charts, scatter plots.

### Schema Explanation

Ask Resona to explain your schema:

> "What is the relationship between the `orders` and `order_items` tables?"

Resona reads the foreign keys, indexes, and column names to give a clear, plain-language explanation.

### Query Optimization

Paste a slow query and ask Resona to optimize it:

> "This query takes 4 seconds. Can you make it faster?"

Resona suggests indexes, rewrites subqueries, and explains the changes.

### Data Exploration

Start with high-level questions and drill down:

> "Which product categories have the highest return rates?"
> "Show me the top 5 countries by revenue this quarter."
> "Are there any users who have never placed an order?"

## How It Works

Resona operates with access to your schema metadata — table names, column names, types, constraints, and foreign key relationships. **It does not send your actual data to AI model providers.** Only the schema structure and your natural language question are sent.

```
You type: "Top 10 customers by revenue this month"
     │
     ▼
Resona reads your schema (table/column names)
     │
     ▼
Sends to AI model: schema context + your question
     │
     ▼
AI returns: SQL query
     │
     ▼
Resona executes query against YOUR database
     │
     ▼
Results + chart displayed to you
```

Your row-level data never leaves your infrastructure.

## Privacy and Security

| What is sent to AI | What stays on your infrastructure |
|--------------------|----------------------------------|
| Table and column names | Actual row data |
| Column types and constraints | Passwords and credentials |
| Foreign key relationships | Query results |
| Your natural language question | Backups and snapshots |

You can optionally **exclude specific tables or columns** from Resona's schema context:

```json
{
  "resona": {
    "schemaExclusions": [
      "public.audit_logs",
      "public.users.password_hash",
      "public.users.ssn"
    ]
  }
}
```

Excluded tables and columns are invisible to Resona entirely.

## Models

Resona supports multiple AI models with different capability and cost tradeoffs:

| Model | Credits per query | Best for |
|-------|------------------|----------|
| GPT-4o mini | 1x | Quick lookups, simple queries |
| GPT-4o | 7x | Complex joins, analysis |
| Claude Sonnet | 14x | Nuanced reasoning, long schemas |
| Free model | 0x | Basic queries (rate-limited) |

## Getting Started with Resona

1. Open the [Data Explorer](/docs/data-explorer)
2. Select a database connection from the sidebar
3. Click **AI Assist** or press `Ctrl+Shift+A`
4. Type your question and press Enter

If this is your first time, Resona will index your schema (takes a few seconds for large databases) and then be ready to answer questions.
