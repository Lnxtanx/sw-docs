# AI Chat

Resona AI Chat lets you have a back-and-forth conversation with your database. Ask questions in plain English, refine your results, and build up complex analyses step by step — without writing a single line of SQL.

---

## Opening AI Chat

### From the Data Explorer

Click the **AI Assist** button in the top-right of the query editor, or press `Ctrl+Shift+A`.

### Standalone Chat Panel

Click **Resona AI** in the main navigation sidebar to open the full-width chat interface.

### Keyboard Shortcut

From anywhere in the app: `Ctrl+Shift+R` (Mac: `⌘⇧R`) opens Resona AI in a floating panel.

---

## Your First Conversation

Start simple. You don't need to mention tables or columns — Resona will figure it out from your schema.

**You:** How many users do we have?

**Resona:**
> Your `users` table has **12,847 records**.
>
> ```sql
> SELECT COUNT(*) AS total_users FROM users;
> ```
> Result: 12,847

**You:** How many signed up this month?

**Resona:**
> **1,234 users** signed up in March 2024.
>
> ```sql
> SELECT COUNT(*) AS new_users
> FROM users
> WHERE created_at >= DATE_TRUNC('month', NOW());
> ```

Notice that Resona remembers the context — you didn't have to say "users" again.

## Iterative Refinement

The best way to use Resona is to start broad and refine. Resona maintains context across the conversation:

> "Show me revenue by country last quarter."

```sql
SELECT
  billing_country,
  SUM(amount_cents) / 100.0 AS revenue
FROM invoices
WHERE paid_at >= '2024-01-01' AND paid_at < '2024-04-01'
GROUP BY billing_country
ORDER BY revenue DESC;
```

> "Great. Now break that down by month."

```sql
SELECT
  billing_country,
  DATE_TRUNC('month', paid_at) AS month,
  SUM(amount_cents) / 100.0 AS revenue
FROM invoices
WHERE paid_at >= '2024-01-01' AND paid_at < '2024-04-01'
GROUP BY billing_country, month
ORDER BY billing_country, month;
```

> "Only show countries with more than $10,000 in revenue."

Resona adds a `HAVING` clause to the previous query.

## Understanding Resona's Responses

Each Resona response includes:

1. **Plain English summary** — what the answer is
2. **The SQL query** — so you can see exactly what ran
3. **Query results** — the actual data
4. **Visualization** — a chart if the data is suitable
5. **Explanation** (optional) — click "Explain this query" for a step-by-step breakdown

### Asking for Explanations

You can always ask Resona to explain a query it generated:

> "Explain what that query is doing."

> "Why did you use a LEFT JOIN instead of INNER JOIN?"

> "What does the PARTITION BY clause do here?"

## Choosing a Model

Click the model selector in the chat header to switch models mid-conversation:

| Model | Speed | Accuracy | Credits/query |
|-------|-------|----------|--------------|
| Free model | Fast | Basic | 0 |
| GPT-4o mini | Fast | Good | 1x |
| GPT-4o | Medium | Excellent | 7x |
| Claude Sonnet | Medium | Excellent | 14x |

> **Tip:** Use **GPT-4o mini** for quick exploratory questions and save **Claude Sonnet** or **GPT-4o** for complex multi-table analysis or when the first attempt doesn't give the right answer.

## When Resona Gets It Wrong

AI-generated SQL isn't perfect. If Resona's query doesn't look right:

1. **Tell it directly:** "That's not quite right — I want orders from this year, not all time."
2. **Be more specific:** Mention exact table or column names if needed.
3. **Copy and edit:** Click **Copy to editor** to move the query to the SQL editor and fix it yourself.
4. **Try a different model:** Some queries are better handled by Claude Sonnet or GPT-4o.

### Reporting a Bad Query

If Resona consistently generates wrong SQL for a type of question, use the thumbs-down button on the response to report it. Your feedback improves future responses.

## Saving AI-Generated Queries

Once Resona generates a useful query, save it to your query library:

1. Click the **Save** button on any query in the chat
2. Give it a name
3. Access it later from **Saved Queries** in the Data Explorer

Saved queries preserve the SQL but not the conversation context.

---

## Conversation History

Resona maintains up to **50 messages** of context in a single conversation. For a fresh start, click **New Conversation** at the top of the chat panel.

Previous conversations are saved in **History** (sidebar) for 30 days.

---

## Limitations

| Limitation | Detail |
|-----------|--------|
| Schema size | Works best with schemas under 500 tables; very large schemas may need guidance |
| Dialect support | Best for PostgreSQL; MySQL and SQLite have slightly lower accuracy |
| Procedural code | Cannot write stored procedures or triggers |
| Cross-database queries | Only queries the selected connection |
| Real-time data | Queries run at request time; no streaming |

---

## Tips for Better Results

- **Name your tables clearly** — `user_subscriptions` is clearer than `usrsubs`
- **Use the right connection** — Make sure the correct database is selected before asking
- **Add date context** — "this quarter" is ambiguous; "Q1 2024" is better
- **Mention units** — "revenue in dollars" vs "revenue in cents" avoids confusion
- **Start simple** — Get a basic query working, then ask Resona to add complexity
