# Workspace & Projects

Resona AI stores everything it generates. The **Workspace** holds exported files, **Projects** organize your analysis sessions, and **Artifact History** gives you a searchable archive of every chart and result ever produced.

![Resona AI interface showing the workspace and project organization panel](/images/data-explorer/resona-ai.png)

---

## Workspace

The Workspace is a persistent file store for reports and exports generated by Resona. It is accessible from the `W` shortcut or the workspace icon in the AI panel toolbar.

### File Types

Resona generates the following file types and saves them to the Workspace automatically:

| Format | How it's generated | Best for |
|---|---|---|
| **PowerPoint (.pptx)** | `generate_ppt` tool | Stakeholder presentations, executive summaries |
| **PDF** | `generate_pdf` tool | Printed reports, email attachments |
| **HTML** | Exported interactive dashboards | Browser-based sharing, embedded reports |
| **Markdown** | `generate_summary_report` tool | Developer-readable summaries, internal wikis |

### Managing Files

Each file in the Workspace can be:

- **Downloaded** — save to your local machine
- **Renamed** — give it a meaningful name for future reference
- **Deleted** — permanently removes the file
- **Versioned** — if Resona generates an updated version of a report in the same session, both versions are kept

Files are scoped to the **Project** that was active when they were created. If you are not using Projects, all files go into the default workspace.

## Projects

Projects let you group related chat sessions and their outputs into a named workspace. When you are investigating a specific question — "Why did revenue drop in March?" or "Analyse user activation rates" — you create a Project for that investigation.

### What a Project Contains

- All chat sessions you assigned to it
- All Workspace files generated during those sessions
- All artifacts from those sessions (accessible in Artifact History filtered to that project)
- A user-defined color and icon for quick visual identification

### Creating and Switching Projects

Open the **Projects** panel from the sidebar folder icon. Click **New Project** and give it a name. To assign a chat to a project, select the project before or after starting the chat.

You can move a chat to a different project at any time from the chat options menu.

## Artifact History

Every artifact Resona has ever generated — across all sessions and projects — is stored in the **Artifact History**. This includes charts, tables, stat blocks, reports, and files.

### Searching and Filtering

The Artifact History panel has a search bar and filter controls:

- **Search** — full-text search across artifact titles and the questions that generated them
- **Filter by type** — show only Charts, Tables, Stat Blocks, Reports, or Files
- **Filter by date** — narrow to a specific time range
- **Filter by project** — show only artifacts from a specific project

### Reusing Artifacts

From Artifact History you can:

- **Open** — re-display the artifact in its full interactive view
- **Copy SQL** — grab the underlying query that generated a data artifact
- **Navigate to chat** — jump back to the original conversation that produced this artifact for more context
- **Pin to Dashboard** — add a historical chart to the live Dashboard without re-running the analysis

This is particularly useful when you remember a useful analysis from weeks ago but can't recall which chat it was in.

## Library

The **Library** is a separate panel for assets you explicitly save for long-term reuse, plus a health overview of your database connections.

### Saved Charts

Any chart or stat block can be saved to the Library by clicking the bookmark icon on the artifact. Saved charts are displayed as a gallery for quick access — these are the metrics and visualizations you check regularly.

### Connection Health

The Library shows the current status of all your connected databases:

- **Healthy** — connection is live and responding
- **Unhealthy** — connection failed the last health check (timeout, auth failure, etc.)
- **Unknown** — not yet checked since the last backend restart

Each connection entry also shows the database size, total table count, and the last time it was used for an analysis.
