# Exploring Your Data

Once you've selected a PostgreSQL connection, the interface gives you a two-panel layout for browsing live data — a sidebar of tables on the left and a high-performance data grid on the right. Connections are created in the [SQL Editor](/database/adding-a-connection) and are available here automatically — no separate setup needed.

> Quick links: [Open Data Explorer](https://data-explorer.schemaweaver.vivekmind.com/) · [Data Explorer Overview](https://docs.schemaweaver.vivekmind.com/data-explorer/) · [SQL Editor](https://sql-editor.schemaweaver.vivekmind.com/)

## Left Sidebar

![The left sidebar showing the schema selector, table search, and table list with row counts](/images/data-explorer/left-sidebar.png)

### Schema Selector

At the top of the sidebar is a **schema dropdown** that lists every schema in the connected database. Changing the schema reloads the table list and clears the current table selection.

- Defaults to `public` for PostgreSQL
- All schemas visible to the connected user are listed
- Switching schemas fetches a fresh table list from the server

### Table List

Below the schema selector, you'll see a searchable list of tables.

**Each table row shows:**

| Element | Description |
|---------|-------------|
| Table icon | Standard table indicator |
| Name | Table name (truncated if long) |
| ⚠️ Warning | Shown if the table has **no primary key** |
| Row estimate | Approximate row count (`47K`, `3.2M`, etc.) |
| AI icon | Hover any table to reveal the Resona AI icon — click it to ask AI about that specific table |

**Search & Filter**

Type in the search box at the top to instantly filter the table list. The count bar updates to show how many tables match (e.g., `3 of 24 tables`).

**Table Selection**

Click any table name to load its data in the right-side grid. The selected table is highlighted with a primary-colored left border and highlighted background.

### Collapsible Sidebar

Click the toggle button in the sidebar header to collapse it to a narrow icon strip (48px), giving the data grid maximum width. Click again to expand.

## Data Grid

The data grid is the heart of the Data Explorer. It renders your table's rows in a high-performance, scrollable table with fixed column headers.

### Column Headers

Each column header displays:

| Element | Description |
|---------|-------------|
| 🔑 Key icon | Gold key icon for primary key columns |
| 🔗 Link icon | Blue link icon for foreign key columns |
| Column name | Bold for PKs, blue-tinted for FKs |
| Type badge | Monospace type label (e.g., `int4`, `varchar?`) — the `?` indicates nullable |
| FK reference | On hover, shows `→ referenced_table` for foreign keys |
| Sort button | Click the sort arrows to cycle through ascending → descending → unsorted |

### Sorting

Click the sort icon on any column header to sort:

1. **First click** — Sort ascending (↑)
2. **Second click** — Sort descending (↓)
3. **Third click** — Remove sort (return to default order)

Sorting resets pagination back to page 1. The active sort column is highlighted.

> Sort operations are performed server-side, so they work efficiently on tables with millions of rows.

## Smart Column Sizing

Column widths are automatically calculated to fit your data using a production-grade sizing engine.

### How It Works

1. **Canvas-Based Measurement** — Column widths are calculated using the browser's `CanvasRenderingContext2D` API with the actual font face and size (`13px Inter`), giving pixel-perfect proportional measurements
2. **95th Percentile Sampling** — Up to 50 rows are sampled per column, and the **95th percentile** width is used. This prevents one abnormally long value from stretching the column far beyond what's useful
3. **Type-Aware Constraints** — Different data types have different width ranges:

| Column Type | Min Width | Max Width |
|-------------|-----------|-----------|
| Integer / Numeric / Boolean | Header width | 160px |
| Timestamp / Date | 180px | — |
| UUID | 270px | — |
| Text / Varchar | 80px | 500px |

### Manual Resizing

- **Drag** the right border of any column header to resize
- **Double-click** the border to **auto-fit** (reset to calculated width)
- Manual widths are **persisted** to `localStorage` and restored when you return

### Persistence & Storage

Manual column widths are saved per-connection, per-schema, per-table. The storage layer features:

- **Debounced Saves** — Disk writes are debounced at 400ms during drag resize to prevent UI jank
- **LRU Cache** — A maximum of 50 saved table configurations are kept. If the limit is exceeded, the least recently used entry is pruned
- **Adaptive Lock Bypass** — If new data requires more than 2.5× the saved width to be readable, the saved width is automatically discarded in favor of the auto-calculated size

## Filtering Data

![The action bar showing the Filter and Columns controls above the data grid](/images/data-explorer/action-bar-filter-columns.png)

The filter system lets you narrow down rows using typed conditions.

### Adding a Filter

1. Click the **Filter** button in the action bar
2. Choose a **column** from the dropdown
3. Choose an **operator**
4. Enter a **value** (press Enter or click "Add Filter")

### Available Operators

Operators adapt based on column type:

| Column Type | Available Operators |
|-------------|-------------------|
| Text | `=`, `≠`, `~ Contains` |
| Number | `=`, `≠`, `>`, `≥`, `<`, `≤` |
| Boolean | `=`, `≠` |
| Date / Timestamp | `=`, `≠`, `>`, `≥`, `<`, `≤` |
| All types | `IN (list)`, `IS NULL`, `NOT NULL` |

### Filter Chips

Active filters appear as tagged chips in the action bar:

```
[ Filter (2) ] | [ status = active ✕ ] [ role = admin ✕ ] Clear
```

- Click the **✕** on any chip to remove that filter
- Click **Clear** to remove all filters at once
- Filter changes reset pagination to page 1

> Filters are applied server-side, ensuring accurate results even on large tables.

## Toggle Column Visibility

Click the **Columns** button in the action bar to show/hide individual columns:

- Each column shows an eye icon (visible) or crossed-out eye (hidden)
- Toggle any column to hide it from the grid
- The button label updates to show the visible count (e.g., `Columns (8/12)`)
- Click **Show all columns** at the bottom to reset

Hidden columns are session-only — they reset when you navigate away.

---

## Refresh Data

Click the **↻ refresh** button in the action bar to re-fetch the current page of data from the database. The icon animates while the request is in progress.

This is useful when:
- Another process has modified the data
- You want to verify that a migration or import succeeded
- You're monitoring live data changes

---

## Click to Copy

Click on any cell in the data grid to copy its value to the clipboard. A brief green **"Copied"** badge appears to confirm. Special values are handled:

| Cell Value | Copied As |
|-----------|-----------|
| `NULL` | The string `NULL` |
| Boolean | `true` or `false` |
| JSON object | Pretty-printed JSON string |
| Date | ISO 8601 string |
| Any other value | Plain string |

## Column Statistics Panel

![Column Statistics Panel showing null percentage, distinct count, min/max values, and top value distribution](/images/data-explorer/columns-stats.png)

Click a **column header** (not the sort icon) to open the Column Stats Panel — a slide-over sidebar that shows live statistics for that column.

### Statistics Displayed

| Stat | Description |
|------|-------------|
| **Type** | Full data type with nullable indicator |
| **Flags** | Primary Key, Indexed badges |
| **Null %** | Percentage of null values |
| **Distinct Count** | Number of unique values |
| **Min / Max** | Minimum and maximum values (for numbers and dates) |
| **Data Completeness** | Visual progress bar showing non-null percentage |

### Top Values

For columns with repeated values, the panel shows the most frequent values with percentage distribution bars. This is useful for:

- Understanding enum-like columns
- Spotting data skew
- Identifying the most common categories

### Sample Preview

A set of sample values is displayed as small tokens, giving you a quick look at what kind of data the column contains without scrolling through the grid.

**Close** the panel by clicking the X button or clicking the same column header again.

## Pagination

The bottom bar shows cursor-based pagination:

```
1–50 of ~47.0K                                    Page 2   ◀ Prev  Next ▶
```

| Element | Description |
|---------|-------------|
| Row range | Current visible range (e.g., `1–50`) |
| Estimated total | Approximate total row count |
| Page indicator | Shown when not on page 1 |
| Prev / Next | Navigate between pages (50 rows per page) |

Pagination is **cursor-based** rather than offset-based, which means:
- Forward pagination is efficient even on tables with millions of rows
- The page counter is approximate (indicated by the `~` prefix)

---

## Row Number Column

The leftmost `#` column displays sequential row numbers relative to the current page. Row numbers automatically account for pagination offset — page 2 starts at row 51.

## Cell Formatting

Different data types are styled for readability:

| Data Type | Display |
|----------|---------|
| `NULL` | Italic gray "NULL" |
| `true` / `false` | Colored pill badges (green/red) |
| Foreign key values | Small arrow indicator (`→`) on hover |
| JSON objects | Serialized to string with truncation |
| Long text | Truncated with ellipsis — click to copy the full value |

## Empty States

The grid handles edge cases gracefully:

| State | What You See |
|-------|-------------|
| No connection selected | "No connection selected" with guidance to choose one |
| No table selected | "Select a table" with arrow pointing to sidebar |
| Table has no rows | "This table is empty" |
| Filters return no results | "No rows match your filters" with a clear link |
| Data load failure | Error message with a **Retry** button |
