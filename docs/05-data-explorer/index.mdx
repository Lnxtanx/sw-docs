# Data Explorer

The Data Explorer is a built-in SQL workbench that lets you connect to your databases, run queries, and explore your data — all inside Schema Weaver. It integrates with Resona AI so you can ask questions in plain English and get runnable SQL back instantly.

![The Data Explorer showing the two-column layout with left sidebar and main data grid](/images/data-explorer/data-explorer.png)

> Quick links: [Open Data Explorer](https://data-explorer.schemaweaver.vivekmind.com/) · [Docs Home](https://docs.schemaweaver.vivekmind.com/) · [Schema Weaver Landing](https://schemaweaver.vivekmind.com/) · [SQL Editor](https://sql-editor.schemaweaver.vivekmind.com/)

---

## What can you do in Data Explorer?

- **Browse live data** — Two-panel layout with a table sidebar and a high-performance data grid.
- **Smart column sizing** — Canvas-measured, 95th-percentile column widths that auto-fit your data.
- **Filter & sort** — Server-side filtering with typed operators and one-click column sorting.
- **Toggle columns** — Show/hide columns to focus on the data that matters.
- **Column statistics** — Click any column header to see null %, distinct count, top values, and more.
- **Click-to-copy** — Click any cell to copy its value to the clipboard.
- **Export data** — Download single tables or entire schemas as CSV, JSON, NDJSON, Excel, or SQL.
- **Full database export** — ZIP archive of every table in the current schema.
- **Write and run SQL** — Full-featured Monaco editor with syntax highlighting and autocomplete.
- **AI-assisted analysis** — Resona AI writes SQL, analyzes tables, generates visual charts, and compiles PowerPoint reports.
- **Agentic Workflows** — Thinking, Tool Execution, and Synthesis phases for deep data exploration.
- **Rich Artifacts** — Interactive tables, charts, stat blocks, and Mermaid diagrams embedded in chat.
- **Workspace Storage** — Persist generated reports, slide decks, and PDF exports.

## Overview of the Interface

The Data Explorer is designed for efficiency, offering a clear view of your data and an integrated AI assistant for deep analysis.

![Resona AI open in the Data Explorer showing the natural-language chat interface with query results and chart output](/images/data-explorer/resona-ai.png)

### Left Sidebar

- **Schema Selector** — Dropdown to switch between database schemas.
- **Table Search** — Filter the table list instantly as you type.
- **Table List** — Scrollable list with row estimates, PK warnings, and AI shortcuts.
- **Resona AI [AI]** — Toggle the integrated agentic chat interface.
- **Dashboard [D]** — View your pinned charts and metrics.
- **Workspace [W]** — Manage persisted files and reports.

### Right Panel — Data Grid

- **Action Bar** — Filter controls, column visibility toggle, refresh, export button, and row count.
- **Column Headers** — Type badges, PK/FK indicators, sortable, resizable, and click for statistics.
- **Row Grid** — Paginated rows with click-to-copy, NULL formatting, boolean badges, and FK arrows.
- **Pagination** — Cursor-based, 50 rows per page, with row range and estimated total.

## Quick Start Workflow

1. Open [Data Explorer](https://data-explorer.schemaweaver.vivekmind.com/) and select a PostgreSQL connection from the connection selector — connections are added from the [SQL Editor](/database/adding-a-connection).
2. Browse the schema tree on the left to find a table.
3. Click the table name to open a quick preview.
4. Use **AI Assist** or the chat interface if you need help writing a query.
5. Use `@table_name` in the AI chat to focus the analysis on specific context.
6. Pin interesting charts to your dashboard for ongoing monitoring.

## Security and Permissions

By default, the Data Explorer operates in a secure **read-only mode**, restricted to `SELECT` statements. This architectural safeguard ensures that analytical exploration never results in accidental data loss or mutations in your production environment.

For environments where data modification is required (such as development or staging), write access can be enabled on a per-connection basis. This allows you to perform `INSERT`, `UPDATE`, and `DELETE` operations directly from the SQL workbench when necessary.

> **Best practice:** Always maintain `read-only` status for production database connections. For heavy analytical workloads, we recommend connecting to a read-replica to avoid impacting primary database performance.

### Row-Level Data Masking

Schema Weaver includes built-in support for **Row-Level Data Masking** to protect sensitive information (PII). When configured for specific tables and columns, sensitive data is automatically redacted before it reaches the interface.

Common masking types include:
- **Email Masking:** Transforms `alice@example.com` into `a***@***.com`.
- **Phone Masking:** Redacts all but the last four digits of a phone number.
- **Full Redaction:** Completely replaces sensitive values with a `[REDACTED]` placeholder.

This ensures that developers and analysts can work with the database structure and non-sensitive data without being exposed to private user information.
