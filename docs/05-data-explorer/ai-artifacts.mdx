# Charts & Analysis Results

When Resona AI finishes an analysis, it outputs **artifacts** — rich interactive components embedded directly in the chat. These are not static images. You can expand them, sort them, pin them to a dashboard, and export their underlying SQL.

> Quick links: [Open Data Explorer](https://data.schemaweaver.vivekmind.com/) · [AI Chat Interface](/data-explorer/ai-chat-interface) · [Workspace & Projects](/data-explorer/ai-workspace)

![Resona AI chat response with an expanded interactive data table artifact](/images/data-explorer/resoan-ai-chat-expand-table.png)

## Data Tables

When Resona runs a SQL query or returns row-level results, it renders them as an interactive data table.

- **Smart formatting** — dates, numbers, and null values are formatted automatically for readability
- **Sortable columns** — click any column header to sort ascending or descending
- **Inline filtering** — type to filter visible rows without re-running the query
- **Export** — download the table as CSV or Excel directly from the artifact toolbar

Data tables are produced by the `run_query`, `sample_rows`, `search_rows`, `compute_top_n`, and `compute_ranking` tools.

## Charts

Resona generates interactive charts using Recharts. The chart type is chosen automatically based on the analysis being performed.

### Bar Chart

Used for comparisons — aggregations, top-N rankings, period-over-period comparisons, segment breakdowns.

Produced by: `compute_aggregation`, `compute_top_n`, `compare_periods`, `compare_segments`, `compute_distribution`

### Line Chart

Used for trends over time — revenue over months, user signups over weeks, moving averages.

Produced by: `compute_trend`, `compute_moving_average`, `forecast_trend`

### Area Chart

Line chart with a filled area — used for cumulative metrics or stacked time series.

### Pie / Donut Chart

Used for proportional breakdowns where parts must sum to a whole.

### Scatter Chart

Used for correlation analysis — plots two numeric columns against each other to show relationships.

Produced by: `compute_correlation` (individual pairs)

### Heatmap

Used for correlation matrices and cohort / retention grids. Cell color intensity represents the value magnitude.

Produced by: `compute_correlation`, `compute_cohort`, `compute_retention`

### Funnel Chart

Used for conversion analysis — each stage shows absolute count and drop-off percentage to the next stage.

Produced by: `compute_funnel`

### Forecast Chart

A line chart extended with a projected future segment and a confidence interval band.

Produced by: `forecast_trend`

## Stat Blocks

Stat blocks are structured analytical outputs for computations that don't fit a standard chart. Each type has its own layout.

### Statistics Block

Summary statistics for one or more numeric columns: mean, median, standard deviation, min, max, 25th/75th/95th percentiles, skewness, kurtosis.

Produced by: `compute_statistics`

### Distribution Block

Frequency histogram showing how values are spread across value buckets.

Produced by: `compute_distribution`

### Seasonality Block

Identifies repeating patterns in time series data. Shows the dominant period (e.g., weekly, monthly), the strength of the pattern, and a visual of the cycle.

Produced by: `detect_seasonality`

### Anomaly Block

Highlights rows or values that deviate significantly from the expected distribution. Each anomaly entry shows the column, the anomalous value, and the expected range.

Produced by: `detect_anomalies`

### Outlier Block

Extreme value detection using IQR and z-score. Separates high and low outliers with their actual values.

Produced by: `detect_outliers`

### Quality Report

Scans a table for data integrity problems: null rates per column, type inconsistencies, duplicate primary keys, and referential integrity violations.

Produced by: `check_quality`

### Text Analysis Block

For string columns: character length distribution, top token frequency, detected pattern types (email, phone, URL, etc.), and example values.

Produced by: `analyze_text_column`

### Cardinality Block

Shows the distinct value count and entropy of a column — useful for understanding how "unique" text or categorical columns are.

Produced by: `compute_cardinality`

### Duplicate Report

Lists groups of rows that are exact or near-duplicate matches with their duplicate counts.

Produced by: `detect_duplicates`

### Regression Block

Linear regression output: feature coefficients with standard errors, p-values, R² score, and a significance assessment per feature.

Produced by: `compute_regression`

### Risk Table

Scores each row against a configurable risk model. Rows are bucketed into risk tiers with a score and the factors that drove it.

Produced by: `classify_risk`

### Table Comparison Block

Side-by-side diff between two tables: schema differences (column adds/removes/type changes) and row-level count and value differences.

Produced by: `compare_tables`

### Pareto Chart

Shows which subset of items (typically 20%) drives the majority of a metric (typically 80%). Combines a bar chart for individual contribution with a cumulative line.

Produced by: `compute_pareto`

## Artifact Interactions

Every artifact — chart, table, or stat block — has a consistent set of actions in its toolbar.

### Expand to Full Panel

Click the **expand icon** to open the artifact in a full-screen overlay. This gives you more room for detailed chart inspection, scrolling large tables, or reading a long report. The expanded view has the same export and action options.

### Pin to Dashboard

Click **Pin** to add the artifact to your Dashboard. The Dashboard is a persistent, grid-based layout where you collect your most important charts and metrics. Pinned artifacts stay on the dashboard across sessions and chat resets.

Access the Dashboard at any time with the `D` shortcut key or the dashboard icon in the toolbar.

![Resona AI interface — pin any chart or table to the Dashboard for persistent access across sessions](/images/data-explorer/resona-ai-@mention-table-overview.png)

### View Underlying SQL

Every data-driven artifact (chart, table, stat block) has a **View SQL** button. This shows the exact SQL query Resona executed to produce the result. You can:

- Copy the SQL to use in the SQL Editor
- Understand exactly what data was fetched and how it was aggregated
- Verify the logic before sharing the result with stakeholders

### Export from Artifact

Individual artifacts can be exported directly:

- **Charts** → Download as PNG image
- **Tables** → Download as CSV or Excel
- **Reports** → Save to Workspace as PDF or PowerPoint (see [Workspace & Projects](/data-explorer/ai-workspace))

## Artifact Placeholder (Streaming)

When Resona is still computing a result, a **skeleton placeholder** appears in the chat in the artifact's final position. This signals that a result is incoming so the layout doesn't jump when the data arrives. Charts and tables stream their data progressively — you often see rows or data points appear as they are computed.
