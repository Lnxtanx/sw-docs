## Export Button

![The Export button and format selection dropdown in the Data Explorer action bar](/images/data-explorer/export.png)

The **Export** button appears in the top-right action bar when you have a connection selected. Click it to open a dropdown with two sections:

1. **Export Current Table** â€” downloads the currently selected table
2. **Export Entire Database (ZIP)** â€” bundles all tables in the active schema into a ZIP archive

## Single Table Export

Select a table in the sidebar, then open the Export dropdown. You'll see the table name with each format option:

| Format | File Extension | Description |
|--------|---------------|-------------|
| **CSV** | `.csv` | Comma-separated values, UTF-8 encoded. Universally compatible with spreadsheets, BI tools, and scripting languages |
| **JSON** | `.json` | Array of objects. Each row is a JSON object with column names as keys |
| **NDJSON** | `.ndjson` | Newline-delimited JSON. One JSON object per line â€” ideal for streaming processors, log tools, and `jq` |
| **Excel** | `.xlsx` | Native Excel spreadsheet. Formatted data with proper column types |
| **SQL** | `.sql` | `INSERT INTO` statements. Ready to replay into another database |

### Export Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| Max rows | 10,000 | Maximum number of rows exported per table |
| Schema | Current schema | The schema of the selected table |

### File Naming

Downloaded files are named with the table name and current date:

```
users_2026-04-04.csv
orders_2026-04-04.xlsx
products_2026-04-04.sql
```

## Full Database Export (ZIP)

To export **all tables** in the current schema at once, click the green **"Export Entire Database (ZIP)"** section in the export dropdown.

### How It Works

1. Click the collapsible "Export Entire Database (ZIP)" header to reveal format options
2. Select a format (CSV, JSON, NDJSON, Excel, or SQL)
3. A toast notification confirms the export has started:

> *Exporting schema 'public'... Preparing ZIP archive of CSV files. This may take a moment depending on database size.*

4. The server bundles every table in the schema into a single ZIP file and streams it to your browser

### ZIP Structure


Each file inside the ZIP follows the same format as a single-table export.

### Timing

Export time depends on:
- Number of tables in the schema
- Total row count across all tables
- Network speed between the server and database
- Chosen format (SQL and Excel are slightly slower than CSV)

> For large databases, the export button shows a loading spinner and disables re-clicking while the ZIP is being prepared.

## API Reference

Exports use direct URL-based downloads. No authentication token is passed in the URL â€” the browser's session cookie handles auth.

### Single Table Export

```
GET /api/data/{connectionId}/tables/{tableName}/export
    ?schemaName=public
    &maxRows=10000
    &format=csv
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `schemaName` | string | `public` | Database schema |
| `maxRows` | number | `10000` | Row limit |
| `format` | string | `csv` | One of: `csv`, `json`, `ndjson`, `excel`, `sql` |

### Full Schema Export

```
GET /api/data/{connectionId}/export-all
    ?schemaName=public
    &format=csv
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `schemaName` | string | `public` | Database schema to export |
| `format` | string | `csv` | One of: `csv`, `json`, `ndjson`, `excel`, `sql` |

Response: ZIP archive streamed as `application/zip`.

## Use Cases

### Data Migration

Export your schema as SQL and replay it in another environment:

```bash
# Export from Schema Weaver, then import
psql -h new-host -d newdb -f users_2026-04-04.sql
```

### Reporting & Analysis

Export as CSV or Excel for use in:
- Google Sheets / Microsoft Excel
- Tableau / Power BI / Metabase
- Pandas / R / Jupyter notebooks

### Backup Snapshots

Use the full database export to create periodic data snapshots:

```bash
# Automate with curl
curl -o backup.zip \
  "https://data.schemaweaver.vivekmind.com/api/data/{connId}/export-all?schemaName=public&format=csv"
```

### Data Processing Pipelines

Export as NDJSON for streaming ingestion:

```bash
# Process with jq
cat users.ndjson | jq 'select(.role == "admin")' > admins.ndjson
```
