## The Editor

The center panel is a fully featured code editor optimized for PostgreSQL schema definitions. It handles large files gracefully and keeps the interface responsive even with hundreds of tables.

The **left sidebar** is your file manager â€” project name, folder tree, open editors list, and file search all live here. Use it to navigate between files, create new ones, and organize your schema across folders.

![The SQL Editor left sidebar showing the project name, Explorer file tree, open editors section, and filter search box](/images/sql-editor/sql-editor-leftside-bar.png)

### Syntax Highlighting

The editor colors your SQL to make it immediately readable:

| Color | Element |
|---|---|
| **Keyword** | `CREATE`, `TABLE`, `PRIMARY KEY`, `REFERENCES`, `NOT NULL`, etc. |
| **Type** | `text`, `uuid`, `timestamptz`, `integer`, `jsonb`, `boolean`, etc. |
| **String** | String literals and quoted identifiers |
| **Comment** | `-- single line` and `/* block */` comments |
| **Number** | Numeric literals and constants |
| **Operator** | `=`, `::`, `->`, `->>`, `@>`, etc. |

### Auto-Indentation

The editor automatically indents new lines inside `CREATE TABLE`, `BEGIN ... END`, and function bodies. Pressing `Enter` after an opening `(` indents the next line by one level.

### Line Numbers

Line numbers are displayed in the left gutter. Clicking a line number selects the entire line.

## Autosave

The editor autosaves your changes automatically. The footer status bar shows:

| Status | Meaning |
|---|---|
| `Autosave âś“` | Autosave is on â€” changes are saved a few seconds after you stop typing |
| `Saving...` | A save is in progress |
| `Saved` + timestamp | The last successful save time |
| `Unsaved changes` | Autosave is off and there are pending changes |

### Toggle Autosave

Click the autosave indicator in the footer to toggle it on or off. When autosave is off, use `Ctrl+S` / `Cmd+S` to save manually.

> **Tip:** Autosave uses optimistic conflict detection. If the server copy was updated by a team member since your last load, a conflict toast appears and lets you choose to overwrite or reload.

## Fullscreen Mode

Click the **fullscreen icon** (â†—) in the top-right of the editor header to expand the editor to fill the entire browser window.

In fullscreen mode:

- The **left Explorer sidebar** appears, giving you the full file tree
- The **bottom status bar** remains accessible
- The ER diagram, compiler, and diff panels open as overlays
- Press `Escape` or click the collapse icon to exit fullscreen

> **Tip:** Fullscreen mode is the recommended way to work on large schemas. The sidebar and status bar give you everything you need without context-switching to other pages.

## Split View

Click the **split view icon** in the header to open a second panel alongside the editor. The right panel shows the **live ER diagram** â€” every table and relationship updates instantly as you type SQL, giving you a real-time visual of your schema without switching views.

Split view is also used for:

- Viewing the diff panel while reviewing changes

Drag the divider between panels to adjust the relative widths.

## The Merge Model

The SQL Editor maintains a **merged view** of all files in the active project. This merged SQL is what feeds the parser, schema compiler, and ER diagram.

```
auth.sql     + billing.sql   + schema.sql
     â†“               â†“               â†“
           Merged SQL (in memory)
                    â†“
     SQL Parser â†’ Schema Compiler â†’ ER Diagram
```

This means:

- You can split your schema across as many files as you like â€” the analysis always sees the full picture
- A `REFERENCES` in `billing.sql` pointing to a table defined in `auth.sql` will resolve correctly
- The order files are merged follows the folder structure (folders first, then files, alphabetically within each group)
- Circular or missing references are caught and reported in the Compiler

## Format Button

The **Format** button in the footer toolbar reformats the SQL in the current file:

- Normalizes indentation and spacing
- Uppercases SQL keywords
- Wraps long lines at 100 characters
- Preserves comments

Formatting is non-destructive â€” it only affects whitespace and casing, never the schema logic.

## Downloading Files

Right-click any file in the Explorer â†’ **Download** to save it as a `.sql` file to your local machine. The download contains the raw SQL content of that file only (not the merged project SQL).

To export the full merged schema, use the **Copy** button in the footer toolbar â€” this copies the entire merged SQL to your clipboard.
