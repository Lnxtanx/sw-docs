# Getting Started

Get Schema Weaver up and running in your local environment with this quick start guide.

## Installation

### Option 1: NPM (Recommended)

Install Schema Weaver as an npm package:

```bash
npm install schema-weaver
```

### Option 2: Docker

Run Schema Weaver in a Docker container:

```bash
docker pull schema-weaver/cli:latest
docker run -it schema-weaver/cli:latest
```

### Option 3: Cloud Platform

Use our hosted platform at [schema-weaver.io](https://schema-weaver.io) (no installation needed).

## Basic Setup

After installing, initialize the setup in your project root:

```bash
npx schema-weaver init
```

This command generates a `schema-config.json` file where you can define your database connections:

```json
{
  "connections": [
    {
      "name": "production",
      "driver": "postgres",
      "url": "postgresql://user:password@localhost:5432/mydb"
    },
    {
      "name": "staging",
      "driver": "postgres",
      "url": "postgresql://user:password@staging.example.com:5432/mydb"
    }
  ],
  "migrationsDir": "./migrations"
}
```

## Connect Your Database

### PostgreSQL

```json
{
  "driver": "postgres",
  "url": "postgresql://username:password@localhost:5432/database_name"
}
```

### MySQL

```json
{
  "driver": "mysql",
  "url": "mysql://username:password@localhost:3306/database_name"
}
```

### SQLite

```json
{
  "driver": "sqlite",
  "url": "file:./database.db"
}
```

## Your First Migration

Create a migration file:

```bash
npx schema-weaver migration create add_users_table
```

This generates a migration file. Edit it:

```sql
-- migrations/001_add_users_table.sql

CREATE TABLE users (
  id BIGSERIAL PRIMARY KEY,
  email TEXT UNIQUE NOT NULL,
  name TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_users_email ON users(email);
```

Preview the migration:

```bash
npx schema-weaver migrate --preview
```

Apply the migration:

```bash
npx schema-weaver migrate apply
```

## Verify Installation

Check that Schema Weaver is working correctly:

```bash
npx schema-weaver status
```

You should see:
- ✅ Schema Weaver version
- ✅ Connected databases
- ✅ Pending migrations
- ✅ Last applied migration timestamp

## Troubleshooting

### "Connection refused"

Ensure your database is running and the connection URL is correct:

```bash
# Test connection
npx schema-weaver validate-connection
```

### "Permission denied"

Your database user needs permissions to:
- Read schema information
- Execute DDL statements (CREATE, ALTER, DROP)
- Read migration history table
